Semantic Elements are Awesome :)

In web development, a semantic html element clearly describes its meaning to both the browser and the developer. In other words, semantic elements have better form and function, providing more purposeful html tags that are easily interpreted by humans and machines alike. It's a best practice nowadays for web developers to use semantic elements, versus having too many nested and unnecessary div elements plastered all over an html document. Non-semantic elements such as div elements, or <div> tags, are basically page dividers that break up web page content into specific sections, but they don't provide any information about the content! These <div> tags have been a quick, go-to element for when web developers need to divide content for styling or structural purposes. Moving forward, it would behoove web developers to make more use of semantic elements since they improve webpage accessibility, searchability, and interoperability.

In review:

Here are some key takeways about semantic elements:

  • Semantic elements clearly describe their meaning to both the browser and the developer.
  • Non-semantic elements, like the aformentioned <div> tell nothing about its content.
  • Make use of semantic elements to enhance the accesbility, searchablity, and interoperability of your web content :) .
